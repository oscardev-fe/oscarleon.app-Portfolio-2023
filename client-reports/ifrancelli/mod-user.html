Módulo Usuario

Registro de usuarios en la Base de Datos

Mostrando datos de usuarios desde la BD

Tareas realizadas

Configurar un Servicio de Cliente HTTP: Se utilizó el servicio HttpClient para realizar solicitudes HTTP a un endpoint que proporciona los datos de la base de datos. Este servicio permite enviar solicitudes GET, POST, PUT, DELETE, etc.

Crear un Endpoint en el Servidor: Endpoint en el servidor que maneje la solicitud para obtener los datos de usuario desde la base de datos. Este endpoint es una API RESTful para devolver los datos requeridos.

Utilizando el servicio HttpClient, se envia una solicitud HTTP al endpoint creado en el servidor. La solicitud GET para obtener los datos de usuario.

Procesar la Respuesta: Una vez que se recibe la respuesta del servidor, se procesa en el sistema. Se deserializa la respuesta JSON en un objeto C# que se utiliza en la aplicación.

Mostrar los Datos al Usuario: Finalmente,se muestran los datos del usuario en la interfaz de usuario.

Funcionamiento API Usuario

Datos técnicos del funcionamiento

Edición de Datos de Perfil:

  • Endpoint: /profile
  • Método: PUT
  • Autenticación: Token de sesión requerido en el header de la petición.
  • Datos de entrada: Los datos del perfil que se desean actualizar.
  • Proceso:
    • Validar el token de sesión.
    • Verificar que el correo electrónico no esté en uso si se desea cambiar.
    • Actualizar los datos del perfil del usuario en la base de datos.
    • Devolver una confirmación de éxito al cliente.
  • Respuesta:
    • Éxito: Un mensaje de confirmación de que el perfil ha sido actualizado.
    • Error: Un mensaje de error si el token es inválido, el correo ya está en uso, o hay un problema con la actualización de la base de datos.
